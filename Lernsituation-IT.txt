VLAN- und IP-Plan (unterschiedliche Subnetzgrößen)
VLAN	Name	Zweck	Netz	Maske	Gateway (Router)
10	Management	IT/Netz-Admin	10.10.10.0/27	255.255.255.224	10.10.10.1
20	Office	Vertrieb/HR/Backoffice	10.10.20.0/24	255.255.255.0	10.10.20.1
30	Dev	Entwicklung/Produktion	10.10.30.0/25	255.255.255.128	10.10.30.1
40	Guest	Gäste/extern	10.10.40.0/26	255.255.255.192	10.10.40.1
Hinweis: Für Einfachheit bleibt Native VLAN = 1 (Default). Best Practice (separates Native VLAN) kann als Erweiterung umgesetzt werden.

Logische Topologie (Beispiel)
S1 (Etage 1, Core/Distribution)
Gi0/1 zu Router G0/0/0 (Trunk)
Fa0/1 zu S2 (Trunk)
Fa0/2 zu S3 (Trunk)
S2 (Etage 2, Access): Endgeräte-Ports als Access in VLAN 20/30/40
S3 (Etage 3, Access): Endgeräte-Ports als Access in VLAN 10/20/40
Arbeitsauftrag (Basis)
VLANs gemäß Tabelle auf allen Switches anlegen und benennen.
Access-Ports je Etage zuordnen (Port-Ranges siehe Konfig).
Trunks konfigurieren:
S1 Gi0/1 ↔ Router G0/0/0 (Trunk)
S1 Fa0/1 ↔ S2 Fa0/1 (Trunk)
S1 Fa0/2 ↔ S3 Fa0/2 (Trunk)
Router-on-a-Stick (Cisco 4331) auf G0/0/0 mit Subinterfaces .10/.20/.30/.40 einrichten.
IPs der PCs passend setzen (Gateway = VLAN-Gateway).
Tests: Pings innerhalb des VLANs und über VLAN-Grenzen (über Router).
Automatische Konfiguration (Copy & Paste für Packet Tracer)
Hinweise:

Namen sind beispielhaft, der Routername wird zufällig vergeben (hier: R-Kiesel).
Auf Switchports ggf. anpassen, falls andere Ports in PT genutzt werden.
Nach Konfiguration: copy running-config startup-config
Router (Cisco 4331) – R-Kiesel
Verkabelung: R-Kiesel G0/0/0 ↔ S1 Gi0/1 (Copper Straight-Through)

          
enable
conf t
hostname R-Kiesel
no ip domain-lookup

interface g0/0/0
 no shutdown

! VLAN 10
interface g0/0/0.10
 encapsulation dot1Q 10
 ip address 10.10.10.1 255.255.255.224

! VLAN 20
interface g0/0/0.20
 encapsulation dot1Q 20
 ip address 10.10.20.1 255.255.255.0

! VLAN 30
interface g0/0/0.30
 encapsulation dot1Q 30
 ip address 10.10.30.1 255.255.255.128

! VLAN 40
interface g0/0/0.40
 encapsulation dot1Q 40
 ip address 10.10.40.1 255.255.255.192

end
wr

        
Switch S1 (Core/Etage 1)
Verkabelung:

S1 Gi0/1 ↔ R-Kiesel G0/0/0 (Trunk)
S1 Fa0/1 ↔ S2 Fa0/1 (Trunk)
S1 Fa0/2 ↔ S3 Fa0/2 (Trunk)
          
enable
conf t
hostname S1
no ip domain-lookup

vlan 10
 name Management
vlan 20
 name Office
vlan 30
 name Dev
vlan 40
 name Guest

! Trunk zu Router
interface gi0/1
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,40
 no shutdown

! Trunk zu S2
interface fa0/1
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,40
 no shutdown

! Trunk zu S3
interface fa0/2
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30,40
 no shutdown

! (optional) Beispiel-Access-Ports EG
interface range fa0/11-12
 switchport mode access
 switchport access vlan 10
interface range fa0/13-18
 switchport mode access
 switchport access vlan 40

end
wr
